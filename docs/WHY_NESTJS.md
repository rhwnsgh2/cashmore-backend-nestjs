# 왜 NestJS인가?

## 핵심: AI 협업을 위한 모듈화

AI로 개발할 때 문제:
- 한 번에 읽을 수 있는 토큰 양 제한
- 거대한 파일, 복잡한 코드 이해 어려움
- 컨텍스트 커질수록 성능 저하

해결책:
- 코드를 독립적인 모듈로 분리
- AI가 한 모듈씩 집중해서 작업 가능

NestJS 선택 이유:
- 모듈 시스템이 프레임워크에 강제되어 있음
- 구조가 정해져 있어서 예측 가능

```typescript
@Module({
  controllers: [UsersController],
  providers: [UsersService],
})
export class UsersModule {}
```

## 다른 프레임워크와 비교

| 프레임워크 | 모듈 시스템 | AI 협업 |
|-----------|------------|--------|
| **NestJS** | 내장, 강제 | 좋음 |
| Express | 없음 | 별로 |
| Fastify | 플러그인 | 중간 |

## AI 협업 예시

- "UsersModule 만들어줘" → users/ 폴더만 보면 됨
- "결제 기능 추가해줘" → orders/ 폴더만 집중
- "서비스 연결해줘" → import 한 줄 추가

## 추가 장점

- **타입스크립트** - AI가 구조 파악 쉬움
- **데코레이터** - 코드가 선언적이라 의도 파악 쉬움
- **의존성 주입** - 의존 관계 명시적
